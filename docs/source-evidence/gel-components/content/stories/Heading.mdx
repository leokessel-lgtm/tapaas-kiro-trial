## About this component

Heading components create a meaningful visual hierarchy on a simple content page.
They have inbuilt spacing with top and bottom margins. If needed, you can override the inbuilt spacing.

There are different standard heading sizes for large screens and smaller screens.

### Changing heading size

On a standard content page, a `h1` should use the Heading size `1` style, `h2` should use the Heading size `2` style, etc.
Although the style may be changed to create larger or smaller headings to create the correct visual hierarchy on the page.

## Accessibility

HTML headings must be used [semantically](https://gel.service.nsw.gov.au/gel-library/foundations/accessibility-overview/semantic-html).
This means `h1` for the main page heading, `h2` for subordinate headings, etc.

### Changing heading level

In some cases, such as headings within components, the heading level may not be correct semantically.
The option of variable heading levels has been provided through using the `headingElement` prop. This is to ensure that your document structure is accessible.

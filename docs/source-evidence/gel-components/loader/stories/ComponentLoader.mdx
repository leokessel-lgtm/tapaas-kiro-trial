# Loader

<Subtitle>
    Loaders tell users that data is loading.
</Subtitle>

## About this component

Loaders signal to users that something is processing. For example, loading an app or saving an update.

The Loader has a transparent overlay. This overlay obscures the page content. It spins until content has loaded.

## Examples

There are 2 types of loaders:
1. Small loader.
2. Large loader.

### Small loader

The small loader component doesn't block an entire screen. The loader and loader label are centred in the parent container.

Use it when smaller items on a screen need to load. For example, cards or accordions.

Do not use it if the full screen needs to load.

<Canvas of={ComponentLoaderStories.Default} />

### Large loader

The large loader component blocks an entire screen. The loader and loader label are centred on the screen when prop `fullPage` is `true`.

Use it when all screen content needs to load.

Don’t use it when the user needs to interact with other areas of the screen.

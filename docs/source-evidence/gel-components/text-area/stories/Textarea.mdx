## About this component

This component is also known as multi-line text inputs or text fields. It's used when someone needs to enter an amount of text that’s longer than a single line.

The size of the textarea can be set to indicate to user the amount of text we expect them to enter.

## Best practices

<PanelTwoColumnContainer>
    <Panel title='Do' type={PanelType.SUCCESS}>
        <ul>
            <li>
                Use an appropriate size to suggest the expected entry length to
                the user.
            </li>
            <li>
                Use a character counter instead of automatically cropping pasted
                text.
            </li>
        </ul>
    </Panel>
    <Panel title="Don't" type={PanelType.ERROR}>
        <ul>
            <li>
                Use for short or single line text entries. Instead, consider
                using [text input](./?path=/docs/text-input--docs).
            </li>
            <li>
                Overuse this component. Users can be deterred from completing an
                action or form if there are excessive textareas.
            </li>
            <li>
                Use a textarea where you can use a combination of other
                elements.
            </li>
        </ul>
    </Panel>
</PanelTwoColumnContainer>

## Usage

It is recommended to wrap Textarea within the Field component. The Field component will provide a standard look and feel to the label, help text and error message associated with form input elements.

- The label identifies the individual text input and lets the user know what is required.
- Help text provides further details on what is needed and is optional.
- The error message lets the user know what the error is and how to fix it.

Find out more about [Field](./?path=/docs/field--docs).

While we do not recommend disabling the textarea as this makes the field invisible to screen reader users, the option is available to use. ### Controlled vs uncontrolled

The Textarea component supports both controlled and uncontrolled usage, depending on how you want to manage state.

#### Controlled

In controlled mode, use `value` and `onChange` props to manage the text content via your own state. The `onChange` prop triggers when a user types in the textarea.

<Canvas of={stories.Controlled} />

#### Uncontrolled

In uncontrolled mode, the component manages its own internal state. Use `defaultValue` to set the initial text content.

<Canvas of={stories.Uncontrolled} />

# Select

<Subtitle>A compact way of displaying multiple choices to the user.</Subtitle>

## About select

Select is also known as a dropdown or dropbox. It’s a standard HTML element.

Select displays a list of options from which a user can select one option. Consider using this component when there’s limited design space.

Research has shown that some people have difficulty navigating and using the select component. If you include it in your designs, consider testing the usability and accessibility with your users.

## Usage

It is recommended to wrap Select within the Field component. The Field component will provide a standard look and feel to the label, help text and error message associated with form input elements.

- The label identifies the individual text input and lets the user know what is required.
- Help text provides further details on what is needed and is optional.
- The error message lets the user know what the error is and how to fix it.

Find out more about [Field](./?path=/docs/field--docs).

### Controlled vs uncontrolled

The Select component supports both controlled and uncontrolled usage, depending on how you want to manage state.

#### Controlled

In controlled mode, use `value` and `onChange` props to manage the input value via your own state. The `onChange` prop triggers when a user types in the input field.

<Canvas of={stories.Controlled} />

#### Uncontrolled

In uncontrolled mode, the component manages its own internal state. Use `defaultValue` to set the initial input value.

<Canvas of={stories.Uncontrolled} />

### Placeholder

The `placeholder` prop adds helpful context text to guide users when no option is selected. It defaults to "Select" if not specified.

To remove the placeholder, pass an empty string `placeholder=""`. When removed, users will see the first actual option as the default selection.

<Canvas of={stories.Placeholder} />

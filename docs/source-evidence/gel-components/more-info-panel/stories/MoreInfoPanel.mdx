# More info panel

<Subtitle>
    Gives the user the option to get in-context details on a specific word,
    phrase or topic.
</Subtitle>

## About this component

The More info panel is a user-initiated modal. It gives additional and in-context information on a specific word, phrase or topic.

Users can choose to interact with the panel by selecting the panel trigger element.

While the panel is open, users cannot interact with the content on the main page and a background area covers it. This helps the user to focus on the information within the panel.

When interacting with the panel, a user does not need to action the information. If you need a user to act, consider using a [Non-dismissible modal](./?path=/docs/modals--docs#non-dismissible).

## Usage

Place the panel trigger element after the content related to information within the panel. It gives users access to more information about the specific topic. Ensure the placement of the trigger stands out.

Ensure the keywords and trigger appear at the end of or after a:

- sentence
- paragraph
- form field.

You may want to add a space if placing the trigger after a sentence. This makes it easier for users to scan, digest and select the element.

### With visible supporting text

We recommend using a functional icon with visible text. For more information, go to [UI icons - Text alternatives](./?path=/docs/ui-icons--about-ui-icons#when-to-use).

- gives the icon a specific meaning
- increases the clickable area
- provides a voice command for people using screen readers.

We suggest using a linked text element before the Info icon.

Other triggers may be used. When selecting a trigger, consider the impact on the user experience.

## Accessibility

The More info panel can create a disruptive user experience by obscuring content on the main page.

Like accordions, the More info panel hides information. Before using a More info panel, test how your content works without one.

### Keyboard and focus interactions

#### Triggering the panel

When a more info panel is triggered, the user’s focus is shifted to inside the more info panel component.
The focus is trapped until a user exits the panel. The user cannot interact with any other element on the page.
The page is greyed out so the user can interact with the panel without distraction.

#### Returning to page

When the panel is closed, the user’s focus is returned to the trigger element.
This ensures context is not lost after interacting with the more info panel.

For more information, go to [2.4.3: Focus Order (Level A) - W3C](https://www.w3.org/WAI/WCAG22/Understanding/focus-order.html).

### Hiding background content from screen readers

To prevent screen readers from accessing content behind the more info panel, you should manually apply `aria-hidden="true"` to the main page content while the more info panel is open.

We recommend wrapping your page content in a single container element (e.g. a `<div>`). When the more info panel is shown, toggle `aria-hidden=”true”` on this container to hide the content.

> Important: The more info panel should not be inside the element with `aria-hidden="true"`, or it will also be hidden from screen readers.

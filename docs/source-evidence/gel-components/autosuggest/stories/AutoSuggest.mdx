## About this component

When using autosuggest, a user can start typing and get returns based on their typed criteria. Sometimes, this component is also known as:

- Autocomplete.
- Combo box.
- Type-ahead.

Once a user has their results, they can then select the appropriate option from what's displayed.

## Default

The default implementation of Autosuggest, where only `suggestions` and `onSelect` are required props.

Wrap Autosuggest in the [`Field`](./?path=/docs/field--docs) component, as you would any other form input from the component library, to provide neccessary accessibility features.

The default filtering logic will search all suggestions based on the input value, using [`String.prototype.startsWith()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/startsWith). Please refer to the 'Custom Suggestions Shape' example below to see how this logic can be extended for more complex filtering and suggestions data.

Focus on the input to view all suggestions, and then start typing to filter them.

## Reset Selection

The Autosuggest component can be reset by simply passing an empty string into the `value` prop. This will reset the internal `value`, `suggestion` and `method` states, and trigger the `onChange` and `onSelect` callbacks.

## Custom suggestions

The Autosuggest component is able to handle more complex suggestions structures, such as an array of objects with multiple key/value pairs. Click 'Show code' below to view an example.

In order for the component to know exactly which value(s) to filter by, render in the suggestions dropdown, and return when a user selects a suggestion, three additional functions must be passed in as props.

1. `filterCustomSuggestions`
    - Handles the filtering of suggestions.
    - Exposes the input `value`, as typed by the user.
    - Exposes the `reason` for filtering suggestions:
        - 'input-changed' - user typed something.
        - 'input-focused' - input was focused.
        - 'escape-pressed' - user pressed Escape to clear the input.
        - 'suggestions-revealed' - user pressed Up or Down to reveal suggestions.
        - 'suggestion-selected' - user selected a suggestion.
    - **This function must return an array of filtered suggestions.**
1. `renderCustomSuggestion`
    - Handles the rendering of each individual suggestion, to display them in the dropdown list.
    - Exposes the `suggestion` which, depending on what has been passed in via the `suggestions` prop, is either a string or an object.
    - **This function must return a string.**
1. `getCustomSuggestion`
    - Handles updating of the input value when a suggestion is clicked or navigated to by up/down keys.
    - Exposes the `suggestion` which, depending on what has been passed in via the `suggestions` prop, is either a string or an object.
    - **This function must return a string.**

The selected suggestion returned via the various callback functions will always include the complete array item. Therefore if your suggestions is an array of objects, the selected object will be returned in-full.

## Appended suggestion

The Autosuggest component allows an additional, custom suggestion to be appended to the suggestions dropdown. This could be used to display a call-to-action within the suggestions dropdown.

This custom suggestion must adhere to the following shape: `{ label: string | node, onSelect: func }`.

## Minimum search characters

The Autosuggest component can be initialised to only show suggestions after a given number of characters have been entered by the user.

If `minSearchChars` is either excluded or set to `0`, then the suggestions will display upon input focus.

# Content design in Kiro specs

Content design must be included across requirements, design and tasks.

## requirements.md

Include content outcomes in user stories and acceptance criteria.

Content outcomes should explain what the user needs to:
- understand
- decide
- do next
- provide
- avoid
- recover from when something goes wrong

## design.md

Define the content model before implementation.

For each page or screen, document:
- page purpose
- heading purpose
- body content purpose
- primary call to action
- secondary actions
- form labels
- hint text
- help text
- error states
- alert or notification content
- confirmation message
- next steps
- content risks
- readability target

## tasks.md

Make content work visible as tasks.

Include tasks for:
- draft page content
- draft component microcopy
- review labels and hint text
- review error messages
- check link purpose
- check readability
- confirm risky source facts
- complete content QA before handoff
